# 인벤 벨가르딘 공략 감시기

로스트아크 인벤 자유게시판·팁과 노하우 게시판에서 벨가르딘 관련 신규 글을 감지해,
Claude AI가 정보성 점수(0~10)와 3줄 요약을 붙여 디스코드로 전송합니다.
10추/30추 모음은 자유게시판의 필터 뷰라 별도로 긁지 않고, 추천수 태그(🔥10추+/🔥30추+)로 표시합니다.

## 1. 디스코드 웹훅 발급 (1분)

1. 디스코드 서버 → 알림 받을 채널 → 채널 편집(톱니바퀴)
2. 연동 → 웹후크 → 새 웹후크 → 이름 지정 → **웹후크 URL 복사**
3. 권장 채널 구조: `#벨가르딘-수집`(봇 자동 전송) + `#벨가르딘-공략`(사람이 검증한 글만 포워드)

## 2-A. GitHub Actions로 돌리기 (권장 시작점)

1. **Private 저장소** 생성 후 이 폴더 전체를 push
2. 저장소 Settings → Secrets and variables → Actions → New repository secret
   - `ANTHROPIC_API_KEY` : Claude API 키 (console.anthropic.com에서 발급)
   - `DISCORD_WEBHOOK_URL` : 위에서 복사한 웹훅 URL
3. Actions 탭 → `inven-watcher` → **Run workflow**로 수동 1회 실행
   - 첫 실행은 기존 글을 "본 글"로 기록만 하고 알림을 보내지 않습니다 (알림 폭탄 방지)
4. 이후 10분 간격 자동 실행. 상태(`state.json`)는 저장소에 자동 커밋됩니다.

주의: 스케줄 cron은 GitHub 사정에 따라 12~20분으로 밀릴 수 있습니다.
또한 60일간 저장소에 커밋이 없으면 스케줄이 자동 비활성화됩니다(state 커밋이 있어 보통 문제없음).

## 2-B. 윈도우 PC로 돌리기 (Actions가 차단될 경우)

```
pip install requests beautifulsoup4
set ANTHROPIC_API_KEY=sk-ant-...
set DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...
python inven_watcher.py
```

작업 스케줄러 → 기본 작업 만들기 → 트리거 "매일" + 고급 설정에서 "10분마다 반복" →
동작에 `python C:\경로\inven_watcher.py` 등록. 환경변수는 시스템 환경변수로 등록하면 편합니다.

## 3. 설정 변경 (inven_watcher.py 상단 CONFIG)

| 항목 | 기본값 | 설명 |
|---|---|---|
| `keywords` | `["벨가르딘"]` | 제목 1차 필터. 출시 후 커뮤니티 약칭이 생기면 추가 |
| `min_score` | 자게 6 / 팁게 4 | 이 점수 미만이면 전송 안 함 |
| `recommend_override` | 10 | 추천수가 이 값 이상이면 점수 무관 전송 |
| `first_run_silent` | True | 첫 실행 알림 폭탄 방지 |

## 4. 8/5 전 검증 체크리스트 (중요)

- [ ] **7월 중 미리 가동**해서 GitHub IP가 인벤에 차단되는지 확인
      (로그에 "파싱 결과 0건" 경고가 반복되면 차단 또는 마크업 변경 → `debug_*.html` 확인)
- [ ] 키워드를 임시로 `["아브렐슈드"]` 같은 현재 활발한 단어로 바꿔 전체 파이프라인 리허설
- [ ] 추천수 파싱이 되는지 확인 (안 되면 🔥태그만 빠지고 나머지는 정상 동작)
- [ ] 인벤 robots.txt 정책 확인
- [ ] Claude API 비용 확인: Haiku 기준 글 1건당 수 원 수준, 출시 첫 주 수백 건이어도 소액

## 5. 저작권 관련

인벤 콘텐츠는 무단 전재·복사·배포가 금지되어 있습니다.
이 도구는 본문을 퍼가지 않고 **제목 + AI 요약 3줄 + 원문 링크**만 전송하도록 설계되어 있으며,
이 구조를 유지해 주세요 (요약을 본문 전체 붙여넣기로 바꾸지 말 것).

## 알려진 한계

- 인벤 목록/본문의 HTML 구조는 실제 페이지로 검증하지 못한 상태입니다.
  파서는 CSS 클래스 대신 글 URL 패턴 기반으로 방어적으로 짜여 있어 웬만하면 동작하지만,
  첫 실행 후 로그와 `debug_*.html`로 반드시 확인하세요.
- 제목에 키워드가 없는 글(예: "3관문 이거 왜 안됨?")은 놓칠 수 있습니다.
  출시 후 커뮤니티에서 쓰는 약칭·관문 표현을 keywords에 계속 추가하는 게 현실적 해법입니다.
